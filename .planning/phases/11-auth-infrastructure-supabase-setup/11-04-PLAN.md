  <action>
Modify app/_layout.tsx to initialize auth on app launch:

1. Add imports:
   - import { useEffect } from 'react'
   - import { initializeAuth } from '@/services/authService'

2. In RootLayout component, add useEffect hook:
   - Call initializeAuth() on mount
   - Wrap in try-catch to prevent app crash if Supabase not configured
   - Log warning if init fails (non-blocking)

3. Store unsubscribe return from initializeAuth():
   - Call unsubscribe() in cleanup function
   - This removes auth state listener when component unmounts

Implementation pattern:
```typescript
useEffect(() => {
  let unsubscribe: (() => void) | null = null;

  const initAuth = async () => {
    try {
      unsubscribe = await initializeAuth();
    } catch (err) {
      console.warn('Auth initialization skipped (Supabase not configured):', err);
    }
  };

  initAuth();

  return () => {
    unsubscribe?.();
  };
}, []);
```

4. Add JSDoc comment explaining this restores session on app launch

IMPORTANT: Do NOT block app launch on auth. useEffect runs after render, so splash screen won't be delayed. If Supabase is unreachable, app continues without auth.

Reference 11-RESEARCH.md "Session State Listener" section.

DO NOT modify existing font loading, splash screen, or theme provider logic.
  </action>

  </verify>

  </done>
