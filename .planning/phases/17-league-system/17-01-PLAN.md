### Task 2: League Schema Migration

<task type="auto">
  <name>Task 2: Create league database schema</name>
  <files>supabase/migrations/007_league_system.sql</files>
  <action>
    Create migration 007 with complete league schema:

    1. **league_tiers** (reference table):
       - tier_key TEXT PRIMARY KEY ('bronze', 'silver', 'gold', 'platinum', 'diamond')
       - tier_order INT NOT NULL UNIQUE (1-5)
       - display_name TEXT NOT NULL
       - color TEXT NOT NULL (#CD7F32, #C0C0C0, #FFD700, #E5E4E2, #B9F2FF)
       - symbol TEXT NOT NULL (emoji)
       - Seed all 5 tiers

    2. **league_cohorts** (weekly groups):
       - id UUID PRIMARY KEY
       - tier_key TEXT REFERENCES league_tiers
       - week_start_date DATE NOT NULL
       - created_at TIMESTAMPTZ
       - UNIQUE(tier_key, week_start_date)

    3. **league_memberships** (user participation):
       - id UUID PRIMARY KEY
       - user_id UUID REFERENCES auth.users
       - cohort_id UUID REFERENCES league_cohorts
       - xp_at_start INT NOT NULL DEFAULT 0
       - xp_at_end INT (null until week ends)
       - final_rank INT (null until week ends)
       - promotion_result TEXT ('promoted', 'relegated', 'stayed', null)
       - created_at TIMESTAMPTZ
       - UNIQUE(user_id, cohort_id)

    4. **Alter user_progress**:
       - ADD COLUMN league_tier TEXT DEFAULT 'bronze'
       - ADD COLUMN timezone TEXT DEFAULT 'UTC'

    5. **RLS policies**:
       - Enable RLS on all new tables
       - Users can view own memberships
       - Authenticated can view tier catalog

    6. **League badge seeds** in badges_catalog:
       - 'bronze_member', 'silver_member', 'gold_member', 'platinum_member', 'diamond_member'
  </action>
  <verify>
    <automated>grep -q "CREATE TABLE.*league_tiers" supabase/migrations/007_league_system.sql && grep -q "league_memberships" supabase/migrations/007_league_system.sql</automated>
  </verify>
  <done>Migration file exists with league_tiers, league_cohorts, league_memberships tables and user_progress alterations</done>
</task>
