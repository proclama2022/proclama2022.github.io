# Phase 16: Calendar Sync Plan 01 - Infrastructure & Service

## Goals
- Enable the app to request Calendar permissions.
- Create a `CalendarService` to abstract interactions with the native calendar.
- Allow users to export plant care schedules (watering) to their device calendar.

## Tasks
- [x] **Install Dependencies**: Install `expo-calendar`.
- [x] **Config**: Add `NSCalendarsUsageDescription` and `NSRemindersUsageDescription` to `app.json` for iOS.
- [x] **Service**: Create `services/calendarService.ts`.
    - `requestPermissions()`
    - `createCalendar()` (or find existing one)
    - `addEvent(title, startDate, endDate, notes)`
    - `getCalendars()`
- [x] **UI Integration**:
    - Add a "Sync to Calendar" button in `Settings` or `PlantDetail`.
    - For MVP, let's add a global toggle in `Settings` -> "Integrations" section, or a per-plant option. Let's start with a simple "Test Sync" or "Sync Next Watering" in Settings to verify.
    - Actually, a better UX is in the **Plant Detail** screen: "Add reminders to Calendar".

## Status
Completed. Added `CalendarService`, integrated "Connect Calendar" in Settings, and "Add to Calendar" in Plant Detail (Care tab).

## Technical Details
- **Library**: `expo-calendar`
- **Permissions**: Requires run-time permission request.
- **Android**: Writes to the default calendar.
- **iOS**: Writes to a custom "Plantid" calendar or the default one.

## Verification
- Run on simulator (Calendar features might be limited on iOS simulator without iCloud, but basic event creation usually works or returns specific errors).
