Create the light meter service with Android native sensor support:

1. Import from 'expo-sensors':
   ```typescript
   import { LightSensor } from 'expo-sensors';
   ```

2. Implement `checkLightSensorAvailability()`:
   - Call `LightSensor.isAvailableAsync()`
   - Return boolean indicating availability
   - Handle errors gracefully (return false)

3. Implement `startLightReading(callback, options?)`:
   - Check platform: if iOS, immediately throw/error (iOS not supported here)
   - Check availability first
   - Set update interval: `LightSensor.setUpdateInterval(100)` (100ms)
   - Add listener with `LightSensor.addListener(({ illuminance }) => {...})`
   - Apply smoothing algorithm (moving average of last 5 readings)
   - Call callback with smoothed LuxReading object
   - Return cleanup function

4. Implement `stopLightReading()`:
   - Remove all listeners: `LightSensor.removeAllListeners()`
   - Reset internal state

5. Implement smoothing:
   - Keep array of last N raw readings (N=5)
   - Calculate moving average
   - Weight recent readings slightly higher (optional enhancement)

6. Error handling:
   - Wrap all sensor calls in try/catch
   - Return meaningful error messages
   - Handle case where sensor becomes unavailable mid-reading

Follow service patterns from services/plantnet.ts and services/profileService.ts.
