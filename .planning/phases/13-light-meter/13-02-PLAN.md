## Technical Context

From RESEARCH.md:
- iOS does NOT expose ambient light sensor to third-party apps (privacy restriction)
- Must use camera-based brightness estimation
- Recommended: Camera preview brightness analysis (Option A)
- Calibration against known reference points critical for accuracy
- Auto-stop after 30 seconds to save battery

### Calibration Approach
1. User places white paper as reference
2. App captures camera brightness at reference point
3. Stores calibration offset locally (AsyncStorage)
4. Applies offset to future readings

### Luminance to Lux Mapping (Initial)
```typescript
const luminanceToLux = (luminance: number): number => {
  if (luminance < 50) return luminance * 2;
  if (luminance < 150) return luminance * 4;
  return luminance * 8;
};
```

- `expo-camera` - may need installation

     const baseLux = luminance < 50 ? luminance * 2

   - Remove calibration from AsyncStorage

7. Helper: `calculateAverageLuminance(pixelData: Uint8Array): number`

   - Calculate average brightness (R+G+B)/3 per pixel
   - Return normalized 0-255 value

Handle errors gracefully, log warnings for debugging.
