# Plan 12-02: Avatar Upload & Profile Services

**Goal:** Implement avatar upload with compression and profile CRUD operations using expo-image-picker and expo-image-manipulator.

## Tasks

### Task 1: Create avatar upload service

Create `services/avatarService.ts`:

**Type definitions:**
```typescript
interface AvatarUploadResult {
  publicUrl: string;
  filePath: string;
}

type AvatarSource = 'gallery' | 'camera';

interface AvatarUploadOptions {
  source: AvatarSource;
  maxSize?: number; // Default 2MB
}
```

**Export functions:**

**`uploadAvatar(userId: string, options: AvatarUploadOptions): Promise<string | null>`**
Algorithm:
1. Request media library permissions using `ImagePicker.requestMediaLibraryPermissionsAsync()`
2. Launch image picker or camera based on `options.source`:
   - Gallery: `ImagePicker.launchImageLibraryAsync()`
   - Camera: `ImagePicker.launchCameraAsync()`
3. If user cancels, return null
4. Crop to square: `aspect: [1, 1]`, `allowsEditing: true`
5. Compress image using `ImageManipulator.manipulateAsync()`:
   - Max dimension: 1200px (match migrationService pattern)
   - Quality: 0.7
   - Format: JPEG
6. Check file size ≤ 2MB (throw error if exceeded)
7. Upload to Supabase Storage `avatars` bucket:
   - File path: `${userId}/avatar_${Date.now()}.jpg`
   - `upsert: true` (replace existing avatar)
   - `cacheControl: '3600'`
8. Return public URL with transformations:
   - Transform: `width: 200, height: 200, resize: 'cover'`

**Error handling:**
- Permission denied: Throw "Photo library permission required"
- File too large: Throw "Image exceeds 2MB limit"
- Upload failed: Throw "Failed to upload avatar"

**Helper function:**
- `compressImage(uri: string): Promise<string>` - Internal function for compression
  - Use `ImageManipulator.manipulateAsync()` with resize action
  - Return compressed URI

### Task 2: Create profile CRUD service

Create `services/profileService.ts`:

**Type definitions:**
```typescript
import { Profile, ProfileWithStats } from '@/types/profile';

interface ProfileUpdateData {
  display_name?: string;
  bio?: string;
  avatar_url?: string;
}

interface ServiceResult<T> {
  success: boolean;
  data?: T;
  error?: string;
}
```

**Export functions:**

**`fetchProfile(userId: string): Promise<ServiceResult<ProfileWithStats>>`**
Algorithm:
1. Call `getProfileWithStats(userId)` from `lib/supabase/profiles.ts`
2. On success: Return `{ success: true, data: profile }`
3. On not found: Return `{ success: false, error: 'Profile not found' }`
4. On error: Return `{ success: false, error: error.message }`

**`updateProfile(userId: string, updates: ProfileUpdateData): Promise<ServiceResult<Profile>>`**
Algorithm:
1. Validate inputs:
   - `display_name` max 50 chars (throw if exceeded)
   - `bio` max 500 chars (throw if exceeded)
2. Call `updateProfile(userId, updates)` from `lib/supabase/profiles.ts`
3. On success: Return `{ success: true, data: updatedProfile }`
4. On error: Return `{ success: false, error: error.message }`

**`uploadAvatarAndUpdateProfile(userId: string, source: AvatarSource): Promise<ServiceResult<string>>`**
Algorithm:
1. Call `uploadAvatar(userId, { source })` from `avatarService.ts`
2. If upload fails, return error result
3. If upload returns null (user cancelled), return `{ success: true, data: '' }`
4. Call `updateProfile(userId, { avatar_url: publicUrl })`
5. Return success with public URL

### Task 3: Enhance profileStore with service actions

Update `stores/profileStore.ts` with service integration:

**Add actions:**
```typescript
interface ProfileState {
  // ... existing state ...

  // NEW: Service actions
  fetchCurrentProfile: (userId: string) => Promise<void>;
  updateProfile: (updates: { display_name?: string; bio?: string }) => Promise<void>;
  uploadAvatar: (source: 'gallery' | 'camera') => Promise<void>;
  refreshStats: () => Promise<void>;
}
```

**Implement actions:**

**`fetchCurrentProfile(userId: string)`**
Algorithm:
1. Set `isLoading: true`, `error: null`
2. Call `profileService.fetchProfile(userId)`
3. On success: Set `currentProfile: data`
4. On error: Set `error: error`, keep current profile
5. Finally: Set `isLoading: false`

**`updateProfile(updates)`**
Algorithm:
1. Get `userId` from `useAuthStore().user.id`
2. Set `isLoading: true`, `error: null`
3. Call `profileService.updateProfile(userId, updates)`
4. On success: Update `currentProfile` with merged data
5. On error: Set `error: error`
6. Finally: Set `isLoading: false`

**`uploadAvatar(source)`**
Algorithm:
1. Get `userId` from `useAuthStore().user.id`
2. Set `isLoading: true`, `error: null`
3. Call `profileService.uploadAvatarAndUpdateProfile(userId, source)`
4. On success: Update `currentProfile.avatar_url` with new URL
5. On error: Set `error: error`
6. Finally: Set `isLoading: false`

**`refreshStats()`**
Algorithm:
1. Get `userId` from `useAuthStore().user.id`
2. Call `fetchCurrentProfile(userId)` to re-fetch stats
3. Updates stats in `currentProfile`

### Task 4: Add profile initialization to app root

Update `app/_layout.tsx` (or `app/(tabs)/_layout.tsx`):

**On app mount (if authenticated):**
1. Check if `useAuthStore().user` exists
2. If user exists and `useProfileStore().currentProfile` is null:
   - Call `useProfileStore().fetchCurrentProfile(user.id)`
3. Handle errors silently (don't block app launch)

**Use `useEffect` with auth user dependency:**
```typescript
const user = useAuthStore().user;
const fetchCurrentProfile = useProfileStore().fetchCurrentProfile;

useEffect(() => {
  if (user && !currentProfile) {
    fetchCurrentProfile(user.id).catch(err => {
      console.error('Failed to fetch profile:', err);
    });
  }
}, [user?.id]);
```
