<task type="auto">
  <name>Create watering service with business logic</name>
  <files>services/wateringService.ts</files>
  <action>
Create services/wateringService.ts with the following exports:

1. markAsWatered(plantId: string, notes?: string): Promise&lt;void&gt;
   - Get plant from usePlantsStore.getState().getPlant(plantId)
   - Create waterEvent: { date: new Date().toISOString(), notes }
   - Update plant: { lastWatered: waterEvent.date, waterHistory: [...plant.waterHistory, waterEvent] }
   - Return waterEvent for notification scheduling

2. getNextWateringDate(plantId: string): Date | null
   - Get plant from store, then care info from careDB.getCareInfo(plant.scientificName)
   - If no care info, return null
   - Use local timezone arithmetic: new Date(year, month, date + waterFrequencyDays)
   - Return Date object (NOT ISO string) for notification trigger

3. getComplianceRate(plantId: string, days: number = 7): ComplianceData
   - Get plant and care info
   - Calculate expectedWaterings: Math.floor(days / care.waterFrequencyDays)
   - Filter waterHistory for last N days (local timezone comparison)
   - Calculate rate: Math.round((actualWaterings / expectedWaterings) * 100)
   - Return { rate, watered: actualWaterings, expected: expectedWaterings, streak: 0 }

4. getWateringStreak(plantId: string): number
   - Sort waterHistory by date descending
   - Iterate from most recent, count consecutive events within (frequency * 1.5 days)
   - Allow 50% margin per research recommendation

5. generateMarkedDates(plant: SavedPlant): Record&lt;string, any&gt;
   - Generate last 30 days as YYYY-MM-DD strings
   - For each date: check if watered (green dot), missed (red dot), or future (gray dot)
   - Return react-native-calendars markedDates object

All functions must handle null/undefined gracefully (return empty/zero values if plant or care info missing).
  </action>
  <verify>grep -E "export.*markAsWatered|export.*getNextWateringDate|export.*getComplianceRate|export.*getWateringStreak|export.*generateMarkedDates" services/wateringService.ts returns 5 matches</verify>
  <done>All watering service functions export correctly, handle missing data gracefully, use local timezone for date math</done>
</task>

<task type="auto">
  <name>Create notification service with platform-specific triggers</name>
  <files>services/notificationService.ts</files>
  <action>
Create services/notificationService.ts using expo-notifications with platform-specific logic per RESEARCH.md Pattern 1:

1. Configure notification channel (Android) and handler (iOS) in module init:
   - setNotificationHandler: { showNotification: true, playSound: true }
   - Android: create channel "watering-reminders" with description

2. schedulePlantNotification(plantId: string, plantName: string, nextDate: Date): Promise&lt;string&gt;
   - Build platform-specific trigger:
     * Android: { type: 'daily', hour: nextDate.getHours(), minute: nextDate.getMinutes() }
     * iOS: { type: 'calendar', repeats: true, dateComponents: { hour, minute, day, month } }
   - Schedule notification with title "Watering Reminder", body "{plantName} needs water today"
   - Include data: { plantId } for routing on tap
   - Return notificationId

3. cancelPlantNotification(notificationId: string): Promise&lt;void&gt;
   - Call Notifications.cancelScheduledNotificationAsync(notificationId)

4. requestPermission(): Promise&lt;boolean&gt;
   - Call Notifications.requestPermissionsAsync()
   - Return status.granted

5. checkPermission(): Promise&lt;'granted' | 'denied' | 'undetermined'&gt;
   - Call Notifications.getPermissionsAsync()
   - Return 'granted' if status.granted, else status.ios.status or 'undetermined'

6. cancelAllPlantNotifications(plantIds: string[]): Promise&lt;void&gt;
   - Get all scheduled notifications via Notifications.getAllScheduledNotificationsAsync()
   - Filter for notifications whose content.data.plantId matches plantIds array
   - Cancel matching notifications in batch

DO NOT implement daily digest (WATER-03) in this plan — that's a separate plan.
