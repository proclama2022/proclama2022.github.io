  <name>Task 2: Add celebration cooldown to gamificationStore</name>
  <files>stores/gamificationStore.ts</files>
  <action>
    Add cooldown mechanism to gamificationStore to prevent celebration spam (max 1 celebration per 3 seconds).

    **Implementation:**
    1. Add constant `CELEBRATION_COOLDOWN_MS = 3000`
    2. Add to GamificationState interface:
       ```typescript
       lastCelebrationAt: number | null;
       canTriggerCelebration: () => boolean;
       recordCelebration: () => void;
       ```
    3. Implement in store:
       ```typescript
       lastCelebrationAt: null,

       canTriggerCelebration: () => {
         const { lastCelebrationAt } = get();
         if (!lastCelebrationAt) return true;
         return Date.now() - lastCelebrationAt >= CELEBRATION_COOLDOWN_MS;
       },

       recordCelebration: () => {
         set({ lastCelebrationAt: Date.now() });
       },
       ```
    4. Reset `lastCelebrationAt` in existing `reset()` function

    **Per CONTEXT.md decisions:**
    - Cooldown: 3 seconds between celebrations
    - Use timestamp tracking (not debounce library)

    **Do NOT:**
    - Do not use setTimeout debounce (use timestamp comparison)
    - Do not change existing toast queue behavior
  </action>
