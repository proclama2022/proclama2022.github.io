# Plan 15-01: Weather Infrastructure & UI

## Goal
Set up the core infrastructure for weather integration: Location service, Weather API client, State management, and basic UI components (Widget & Settings).

## Tasks
- [x] **Location Service**: Implement `services/location.ts` with `expo-location`.
    - Handle permissions.
    - Get current coordinates.
    - Reverse geocode (City, Country).
- [x] **Weather Service**: Implement `services/weather.ts` using Open-Meteo API.
    - Fetch current weather + forecast.
    - Map WMO codes to icons/conditions.
- [x] **Weather Store**: Create `stores/weatherStore.ts` (Zustand).
    - Store `data`, `location`, `isLoading`, `error`.
    - Persist with `AsyncStorage` (cache for 1 hour).
- [x] **Weather Widget**: Create `components/WeatherWidget.tsx`.
    - Display Temp, Icon, Location, Rain.
    - Handle loading/error states.
    - Theme-aware styling.
- [x] **Settings Integration**: Add toggle to `app/(tabs)/settings.tsx`.
    - Request permissions on toggle ON.
    - Show current location status.
- [x] **Home Screen Integration**: Add `WeatherWidget` to `app/(tabs)/index.tsx` header.
- [x] **Localization**: Add strings to `en.json` and `it.json`.

## Verification

- [x] Home Screen -> Shows Widget with correct data.
